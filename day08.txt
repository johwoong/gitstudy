커밋 정보 확인하기
	소스트리 --> 브렌치
	cli --> git log
		--graph 옵션: 그래프 모양으로 커밋 보여주기
		--online 옵션: 커밋id와 메시지를 한줄에
		-숫자: 최근 해당 숫자만큼의 커밋 내용 보기

충돌
	같은 파일내에서 같은 줄ㅇ르 수정하거나 삭제하거나 변경한 후, 해당 대상을 하나로 합칠 때 충돌이 발생한다.
	충돌이 발생되면 자동으로 합쳐진 내용이 커밋되지는 않고 직접 내용을 수정한 후에 똑같이 스테이지에 올려주고, commit을 해줘야 한다



re-base
	base를 다시 설정한다.
	base: 브렌치가 뻗어나가기 시작하는 커밋
	
	뿌리를 바꿔주고 싶은 브렌치에서 작업
		git rebase 뿌리가 존재하는 브렌치명
	rebase 완료된 후에는 merge하여 합쳐주는 것도 가능

rebase과정에서 충돌
	충돌 내용을 수정한 후 add 하여 스테이지에 올려주고,
	git rebase --continue를 사용하여 리베이스를 진행한다.
